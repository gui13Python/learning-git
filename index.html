O que é Git?

Git é um sistema de controle de versão distribuído criado por Linus Torvalds em 2005, o mesmo criador do kernel Linux. É uma ferramenta essencial para desenvolvedores que permite rastrear mudanças no código-fonte durante o desenvolvimento de software.

Controle de Versão

Registra todas as alterações feitas no código ao longo do tempo

Colaboração

Permite que várias pessoas trabalhem no mesmo projeto simultaneamente

Histórico

Mantém um histórico completo de todas as modificações

Reversão

Permite voltar a versões anteriores do código quando necessário

Para que serve o Git?

O Git resolve problemas comuns no desenvolvimento de software:

Controle de Mudanças

Evita a perda de código importante, permitindo que você volte a qualquer ponto no histórico do projeto.

Trabalho em Equipe

Facilita a colaboração entre desenvolvedores, mesmo que estejam em locais diferentes.

Experimentação Segura

Permite criar branches (ramos) para testar novas funcionalidades sem afetar o código principal.

Debugging

Ajuda a identificar quando e onde um bug foi introduzido no código.

Para quem o Git é indicado?

O Git é útil para diversos profissionais e situações:

Desenvolvedores

Profissionais que escrevem código regularmente

Estudantes

Universitários e aprendizes de programação

Designers

Que trabalham com arquivos de design versionados

Escritores

Que precisam versionar documentos e livros

Quando começar a usar?

Você deve começar a usar Git quando:

  • Iniciar seu primeiro projeto de programação
  • Trabalhar em equipe em qualquer projeto
  • Precisar manter histórico de alterações
  • Quiser contribuir com projetos open source

Como instalar o Git

O Git está disponível para Windows, macOS e Linux. Siga os passos abaixo para sua plataforma:

Instalação no Windows

  1. Acesse o site oficial: git-scm.com/download/win
  2. Baixe o instalador para sua versão do Windows (32 ou 64 bits)
  3. Execute o arquivo baixado e siga as instruções do instalador
  4. Mantenha as configurações padrão recomendadas
  5. Após a instalação, abra o Git Bash ou Prompt de Comando
  6. Verifique a instalação com o comando: git --version

Instalação no macOS

  1. Via Homebrew (recomendado):
  2. Abra o Terminal
  3. Instale o Homebrew se ainda não tiver: /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  4. Instale o Git: brew install git
  5. Alternativa: Baixe o instalador em git-scm.com/download/mac
  6. Verifique a instalação: git --version

Instalação no Linux

Use o gerenciador de pacotes da sua distribuição:

Ubuntu/Debian

sudo apt update && sudo apt install git

Fedora

sudo dnf install git

Arch Linux

sudo pacman -S git

CentOS/RHEL

sudo yum install git

Após instalar, verifique com: git --version

Configuração Inicial

Após instalar, configure seu usuário:

git config --global user.name "Seu Nome"
git config --global user.email "seu@email.com"

Comandos Git Essenciais

Estes são os comandos mais importantes que você precisa conhecer:

git init

Básico

Inicializa um novo repositório Git em um diretório

git init

git clone

Básico

Clona um repositório existente para sua máquina local

git clone https://github.com/usuario/repositorio.git

git add

Básico

Adiciona arquivos ao staging area (preparação para commit)

git add arquivo.txt
git add .

git commit

Básico

Registra as mudanças no repositório com uma mensagem

git commit -m "Mensagem descritiva"

git push

Intermediário

Envia commits locais para um repositório remoto

git push origin main

git pull

Intermediário

Baixa e integra alterações do repositório remoto

git pull origin main

git branch

Intermediário

Lista, cria ou deleta branches (ramos)

git branch
git branch nova-feature

git merge

Avançado

Combina branches diferentes

git merge nome-da-branch

git status

Básico

Mostra o estado atual do repositório

git status

git log

Básico

Exibe o histórico de commits

git log
git log --oneline

git checkout

Intermediário

Muda entre branches ou restaura arquivos

git checkout nome-da-branch

git reset

Avançado

Desfaz commits ou muda o estado do repositório

git reset --soft HEAD~1

Fluxo de Trabalho Básico

1

2

Dicas para Praticar Git

  • Crie um repositório para um projeto simples (como uma calculadora)
  • Faça pequenas alterações e commite frequentemente
  • Experimente criar e mesclar branches
  • Use o GitHub para hospedar seu código
  • Participe de projetos open source para praticar colaboração
  • Resolva conflitos de merge intencionalmente para aprender
